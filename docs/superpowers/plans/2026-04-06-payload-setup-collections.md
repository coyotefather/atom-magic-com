# Payload CMS Setup & Collections Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install Payload CMS into the existing Next.js project, configure Neon (Postgres) and Vercel Blob, and define all 13 content collections mirroring the current Sanity schemas.

**Architecture:** Payload lives inside the existing `atom-magic-com` Next.js app (no separate service). The admin UI mounts at `/admin`. All collections use Lexical rich text instead of Sanity's markdown plugin. Media uploads go to Vercel Blob via the official Payload plugin.

**Tech Stack:** `payload`, `@payloadcms/next`, `@payloadcms/richtext-lexical`, `@payloadcms/db-postgres`, `@payloadcms/storage-vercel-blob`, Neon (serverless Postgres), Vercel Blob

### Task 3: Configure environment variables

**Files:**
- Modify: `.env.local` (local dev only, gitignored)

- [ ] **Step 1: Add required env vars to `.env.local`**

Add the following to `.env.local` (values come from Neon dashboard and Vercel Blob dashboard — see Step 2):

```
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
PAYLOAD_SECRET=your-random-32-char-secret-here
BLOB_READ_WRITE_TOKEN=vercel_blob_rw_...
```

- [ ] **Step 2: Provision Neon database**

  - `DATABASE_URL` is already set — pulled via `vercel env pull .env.local` after provisioning Neon through the Vercel dashboard.

- [ ] **Step 3: Provision Vercel Blob**

  - In Vercel dashboard → Storage → Create Blob store → Copy `BLOB_READ_WRITE_TOKEN` into `.env.local`.
  - Or: `vercel blob create` via CLI.

- [ ] **Step 4: Generate a secure `PAYLOAD_SECRET`**

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Paste output as `PAYLOAD_SECRET` value.

- [ ] **Step 5: Add env vars to Vercel project** (for deployment)

```bash
vercel env add DATABASE_URL  # already set by Neon integration
vercel env add PAYLOAD_SECRET
vercel env add BLOB_READ_WRITE_TOKEN
```
